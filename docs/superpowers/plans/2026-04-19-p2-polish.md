## Task 6: Add `generateStaticParams` to tag, people, and location slug pages

**Files:**
- Modify: `app/tag/[slug]/page.tsx`
- Modify: `app/people/[slug]/page.tsx`
- Modify: `app/location/[slug]/page.tsx`

**Context:**

All three pages have `export const revalidate = 3600` but no `generateStaticParams`. In Next.js 16 App Router, `revalidate` on a dynamic route has no effect at build time unless `generateStaticParams` is present to seed the paths. Without it, these pages are fully dynamic (SSR per request), and the ISR value only applies to routes that were visited at least once.

Adding `generateStaticParams` causes Next.js to pre-render all known slugs at build time, after which ISR takes over for revalidation. Slugs not returned by `generateStaticParams` will still be rendered on-demand and cached.

**How to find the right API functions:**

Before implementing, read each page file to see which API function it already calls to resolve the slug. The `generateStaticParams` function should call the "list all" version of that same API.

- [ ] **Step 1: Read `app/tag/[slug]/page.tsx` in full**

2. The existing `generateMetadata` function to understand the slug pattern
3. Confirm `revalidate = 3600` is present

Same as Step 1 for people.

- [ ] **Step 3: Read `app/location/[slug]/page.tsx` in full**
