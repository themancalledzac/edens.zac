## Task 8: `useMemo` → `useCallback` in CollectionContentRenderer

**Files:**
- Modify: `app/components/Content/CollectionContentRenderer.tsx`

`useMemo` is the correct primitive for memoizing computed values. `useCallback` is the correct primitive for memoizing functions. Line 88 uses `useMemo` where the result is either `undefined` or a function — `useCallback` is the right call, but since it conditionally returns `undefined`, the pattern needs wrapping.

Read `app/components/Content/CollectionContentRenderer.tsx` lines 88-130 to see the full `handleClick` block. The fix:

- [ ] **Step 1: Replace `useMemo` with `useCallback`**

The current `useMemo` returns a function or `undefined`. The correct pattern for a conditional callback is to use `useCallback` and check conditions inside:

```tsx
// Before (useMemo returning a function or undefined):
const handleClick = useMemo(() => {
  if (contentType === 'TEXT') return;
  if (isReorderMode) return;
  // ...builds and returns a function
}, [contentId, contentType, ...]);

// After (useCallback — always returns a function, guards inside):
const handleClick = useCallback(() => {
  if (contentType === 'TEXT') return;
  if (isReorderMode) return;

  if (_hasSlug && !onImageClick) {
    router.push(`/${_hasSlug}`);
    return;
  }

  const fullScreenContent: ContentImageModel | ContentParallaxImageModel = {
    id: contentId,
    contentType: contentType === 'GIF' ? 'GIF' : 'IMAGE',
    imageUrl: imageUrl || '',
    title: alt,
    orderIndex: 0,
    visible: true,
  } as ContentImageModel;

  createContentClickHandler(
    contentId,
    onImageClick,
    enableFullScreenView,
    onFullScreenImageClick,
    fullScreenContent
  )?.();
}, [
  contentId,
  contentType,
  imageUrl,
  alt,
  onImageClick,
  enableFullScreenView,
  onFullScreenImageClick,
  _hasSlug,
  router,
  isReorderMode,
]);
```

Read the file carefully before making changes — specifically look at how `handleClick` is used downstream (what it's passed to) to ensure the refactor is compatible. If `handleClick` is passed as an optional prop that can be `undefined`, the refactored version (which always returns a function) must still work in that context. If callers check `handleClick && handleClick()`, replace with `handleClick()` since the guards are now inside.

If the refactor is not straightforward due to downstream consumers expecting `undefined`, report as DONE_WITH_CONCERNS with details.
